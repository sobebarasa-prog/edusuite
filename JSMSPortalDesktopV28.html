💾 Local Backup & Restore

Export a full JSON backup of all school data. Import to restore on any device.

💡 Tip: Export backups weekly and store them safely. Import to restore on a new device or after data loss.
☁️ Cloud Backup (Manual)

Export your backup file, then upload it to Google Drive, Dropbox, or any cloud storage manually.

📌 How to back up to the cloud: Tap "Export Backup File" to download the .json file to your device, then upload it to Google Drive, Dropbox, or email it to yourself. To restore, download the file and tap "Import Backup File".
📱 MobiDrive / Other Cloud Storage

Share your backup to any cloud storage app installed on your device (OneDrive, Dropbox, MobiDrive, etc.).

Opens your device's share sheet. Choose any cloud app (MobiDrive, OneDrive, Dropbox, Files, etc.) to save the backup.
⚙️ Supabase Cloud Database ADVANCED

Connect to a Supabase project for real-time cloud sync. All data keys are synced automatically — 8 seconds after any change, every 5 minutes, and on page close.

🔁 Auto-sync is active when configured. Every edit is queued and pushed within 8 seconds. A full sync runs every 5 minutes, on tab focus, and on page close (via sendBeacon). All edu2_* data keys are included — students, fees, attendance, timetable, library, staff, notifications, receipts, stamps, signatures, departments, and more.
🔐 Security note: Your Supabase anon key is stored locally. Use Row Level Security (RLS) on Supabase to protect data. Create a table named school_data with columns: id, school_id, payload, updated_at. Session tokens and UI preferences are never synced.
